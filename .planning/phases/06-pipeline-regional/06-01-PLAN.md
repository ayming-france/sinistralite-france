<task type="auto">
  <name>Task 1: Create parse_regional.py with full extraction pipeline</name>
  <files>data/pipeline/parse_regional.py</files>
  <action>
Create `data/pipeline/parse_regional.py` as a standalone script with `--pdf` and `--out` arguments (default `--out data/regional-data.json`).

**Structure:**

1. **CAISSE_MAP constant** (dict mapping canonical PDF names to stable IDs). Include all 16 metropolitan caisses (CRAMIF + 15 Carsat) plus 5 DOM-TOM (4 CGSS + CSS Mayotte). Use the exact mapping from RESEARCH.md Pattern 5. Add a `type` field per caisse: `"cramif"`, `"carsat"`, `"cgss"`, or `"css"`.

2. **find_tableau_page(pdf, label)** function: Scan all pages for a page containing the label text (e.g., "Tableau 9" or "Tableau 17"). Return `(page_index, page)` tuple. Log the found page number. Raise an error if not found.

3. **merge_multiline_rows(raw_rows)** function: Implement exactly as in RESEARCH.md Pattern 3. A continuation row has `None` for all columns except the first. Append the first column text to the previous row's first column with a space separator. Skip fully-None rows.

4. **normalize_caisse_name(s)** function: Implement as in RESEARCH.md Pattern 4. Collapse whitespace with `' '.join(s.split())`, apply `unicodedata.normalize('NFC', s)`, strip.

5. **parse_fr_number(s)** function: Reuse the same logic from `parse_pdf.py` (replace spaces and nbsp, cast to int). Return `None` on failure instead of raising.

6. **extract_regional_table(page, has_salaries)** function:
   - Call `page.extract_table(TABLE_SETTINGS)` with the settings from RESEARCH.md (vertical_strategy: "lines", horizontal_strategy: "lines", snap_tolerance: 3, join_tolerance: 3, intersection_tolerance: 3, text_x_tolerance: 5, text_y_tolerance: 3).
   - Run `merge_multiline_rows()` on the result.
   - Skip header rows (detect by presence of year strings like "2020", "2021" in the row).
   - Filter out rows where normalized first column matches "Total", "Ensemble", "France", or "France entiere" (case-insensitive).
   - For each remaining row: normalize the caisse name, look up in `CAISSE_MAP`, parse numeric columns with `parse_fr_number()`.
   - The `has_salaries` boolean controls whether to expect a salaries column (True for Tableau 9, False for Tableau 17).
   - Log warnings for unmatched caisse names (not in CAISSE_MAP). Do not fail.
   - Return a list of dicts: `[{"id": str, "name": str, "type": str, "values": {year: count}, "salaries": {year: count} | None}]`

7. **parse_regional_pdf(pdf_path)** function:
   - Open PDF with pdfplumber.
   - Find Tableau 9 page using `find_tableau_page(pdf, "Tableau 9")`.
   - Find Tableau 17 page using `find_tableau_page(pdf, "Tableau 17")`.
   - Extract AT data from Tableau 9 with `has_salaries=True`.
   - Extract Trajet data from Tableau 17 with `has_salaries=False`.
   - Merge results: for each caisse ID, combine AT and Trajet dicts, plus salaries from AT.
   - Build output following the exact JSON schema from RESEARCH.md Pattern 6.
   - Run `validate_output()` before returning.

8. **validate_output(caisses)** function:
   - Assert at least 16 metropolitan caisses (type "carsat" or "cramif").
   - Assert each metropolitan caisse has non-zero AT values for years 2020-2024.
   - Assert each metropolitan caisse has non-zero Trajet values for years 2020-2024.
   - Warn (do not fail) if fewer than 4 DOM-TOM caisses are present.

9. **main()** function with argparse:
   - `--pdf` (required): path to rapport annuel PDF
   - `--out` (default: computed relative to script location, `../../data/regional-data.json` or `data/regional-data.json`)
   - `--dry-run` flag: print extracted caisse names without writing JSON (for debugging caisse name matching)
   - Call `parse_regional_pdf()`, write JSON with `ensure_ascii=False, indent=2`.
   - Print summary: number of caisses, years covered, any warnings.

**Important details:**
- Use `if __name__ == "__main__": main()` guard.
- All print output goes to stderr for logging, JSON output to file only.
- French comments and docstrings with proper accents.
- Handle the case where a column header row has year strings that might confuse the parser: detect header rows by checking if any cell contains a 4-digit year string matching `r"^20\d{2}$"`.
- The year columns in the PDF table may not be labeled with just the year; they might be nested under sub-headers. Use column index detection from the header row rather than hardcoded indices.
