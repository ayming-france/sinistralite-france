<task type="auto">
  <name>Task 2: Implement setupTapPanel logic and close-on-view-switch</name>
  <files>js/map.js</files>
  <action>
  **In `js/map.js`:**

  1. Add `setupTapPanel(svgId, viewType)` function (before the `DOMContentLoaded` block). Logic:
     - Gate on `navigator.maxTouchPoints === 0` → return early (desktop, no tap panel needed)
     - Get `svg` by `svgId` and `panel` by `getElementById('mapTapPanel')`; return if either missing
     - Determine `metricLabel`: `'Accidents du travail'` for `'at'`, `'Accidents de trajet'` for `'trajet'`
     - Add `click` listener on the SVG element:
       - `e.target.closest('[data-caisse]')` to find the caisse group (same pattern as `setupTooltip` and `setupLinkedHighlight`)
       - Look up the caisse in `regionalData.caisses` by ID
       - Get active year via `getActiveYear(viewType)` (already exists in map.js)
       - Get metric value from `caisse[viewType === 'at' ? 'at' : 'trajet'][year]`
       - If null, return
       - Set `.tap-panel-name` textContent to `caisse.name`
       - Set `.tap-panel-metric` textContent to `metricLabel`
       - Set `.tap-panel-val` textContent to `${val.toLocaleString('fr-FR')} (${year})`
       - Add `'open'` class to panel
       - Call `e.stopPropagation()` to prevent the document click handler from immediately closing the panel (Pitfall 3 from research)

  2. Add `closeTapPanel()` function: removes `'open'` class from `#mapTapPanel` if it exists and has the class.

  3. In the `DOMContentLoaded` block (after the existing `setupLinkedHighlight` calls, line 348), add:
     ```js
     setupTapPanel('france-map-at', 'at');
     setupTapPanel('france-map-trajet', 'trajet');
     ```

  4. Add a single document click listener for dismiss-on-outside-tap (inside `DOMContentLoaded`, after the setup calls):
     ```js
     document.addEventListener('click', (e) => {
       const panel = document.getElementById('mapTapPanel');
       if (panel && panel.classList.contains('open') && !panel.contains(e.target)) {
         panel.classList.remove('open');
       }
     });
     ```
     Also add click handler on the `.tap-panel-close` button inside the panel:
     ```js
     const closeBtn = document.querySelector('#mapTapPanel .tap-panel-close');
     if (closeBtn) closeBtn.addEventListener('click', () => closeTapPanel());
     ```

  5. Export `closeTapPanel` so `nav.js` or `app.js` can call it on view switch. Add it to the exports if `map.js` is a module. Alternatively, since `map.js` is `type=module` and not imported by `nav.js` (which is a regular script), use a simpler approach: call `closeTapPanel()` from within `map.js` by listening to `hashchange`:
     ```js
     window.addEventListener('hashchange', () => closeTapPanel());
     ```
     This closes the panel whenever the URL hash changes (which happens on every view switch via `nav.js` `switchView` line 59-62). This avoids cross-module coupling.

  **Important:** Do NOT create a second `DOMContentLoaded` listener. Add all new calls inside the existing one (per anti-pattern from research).
